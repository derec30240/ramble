# Ramble

移动端 AI 对话客户端：会话以 git 式消息树组织，支持分叉、重新生成与树状历史浏览。对话树只记录有效内容，不是操作日志。

## Language

**Provider (提供商)**:
模型服务的连接配置实体：名称、base URL、API key、协议类型。内置预设列表，用户可自由增删改。协议差异由适配层吸收，OpenAI 格式为第一公民。
_Avoid_: 厂商, 平台, 渠道

**Session (会话)**:
一棵完整的消息树及其全部分支历史。
_Avoid_: Chat, Conversation, Thread

**Node (节点)**:
树中的一条消息——用户消息或 AI 回复，两类节点父子交替。系统提示词不是节点。
_Avoid_: Round, Turn, 消息对

**Generation Snapshot (生成快照)**:
附着在每个 AI 回复节点上的配置快照：系统提示词、模型、生成参数。记录该回复生成时刻生效的配置。
_Avoid_: Config, Preset, 预设

**Session Config (会话配置)**:
Session 级当前生效的配置：系统提示词、模型、生成参数。所有新生成（新消息、重新生成、编辑重发）一律使用它。独立于任何生成快照，不随 Checkout 变化；仅当用户显式用某个节点的生成快照覆盖时才改变。模型恒非空——没有模型（选定或默认）时不允许创建会话。
_Avoid_: 当前配置, 全局设置

**Model Default Params (模型默认参数)**:
绑定在 Provider+模型上的默认参数集，选定该模型时作为参数初值，无则参数为空。参数是稀疏集——未显式设置的参数一律不发送；参数永不跨模型携带，换模型即重载新模型的默认参数。
_Avoid_: 通用参数, 全局参数

**HEAD**:
每个 Session 持有的当前位置指针，指向树中某个节点。聊天视图只渲染 root → HEAD 的线性路径；新消息永远从 HEAD 发出并推进 HEAD。
_Avoid_: 当前分支, 活动路径

**Checkout (检出)**:
将 HEAD 移动到树中任意节点的操作，由树状视图中点选节点触发。
_Avoid_: 切换分支, 跳转

**Branch (分支)**:
从根到某个叶子的一条对话线。可选的分支名绑定在叶子上，当 HEAD 在该叶子上延伸时名称自动推进到新叶子；名称不影响祖先节点与派生支线，也不受它们影响。默认无名。
_Avoid_: 书签, Tag

**Node Status (节点状态)**:
AI 回复节点的完成度标记：完成（complete）或终止（stopped，手动停止或连接中断后保留的部分内容）。终止节点是有效内容，可从其继续对话。有效节点的判定标准是正文非空；正文为空的尝试不产生节点。
_Avoid_: 错误节点, 失败节点

**Reasoning Trace (思考内容)**:
AI 节点上与正文分离的独立字段，存储模型的推理轨迹。默认折叠展示，永不参与上下文组装，不计入节点有效性判定。
_Avoid_: 思维链, CoT 内容

**Usage (节点用量)**:
附着在 AI 节点上的 API 报告用量：prompt_tokens（该次生成消耗的完整上下文）与 completion_tokens（本条回复）。只取 API 返回值，永不本地估算；用户节点没有用量。
_Avoid_: token 统计, 字数统计

**Cumulative Usage (累计用量)**:
Session 级只增不减的消费账本：累加每次 API 报告的用量，与树内容无关——删除节点不回退、未报告的尝试记 0。因此是实际消耗的下界，UI 须向用户说明可能少于实际。
_Avoid_: 会话总 token, 用量统计

**Default Session Config (默认会话配置)**:
软件级配置模板，仅含默认系统提示词与默认模型，均可为空；草稿态由它与模型默认参数解析出初始会话配置。仅由用户显式修改——各配置面首次在会话中填写后触发"保存为默认"提示。
_Avoid_: 全局配置, 预设配置

**Draft (草稿态)**:
从进入新建会话界面到发出首条消息之间的状态：无 Session、无节点、不持久化。首条用户消息发送时才原子地创建 Session（虚拟根 + 首个用户节点）。会话列表永无空会话。
_Avoid_: 空会话, 临时会话

**Virtual Root (虚拟根)**:
每个 Session 唯一的结构锚点，所有开场白（首条用户消息及其编辑重发版本）都是它的孩子。不是消息，不渲染、不参与生成、不携带任何配置，仅保证 Session 恒为单根的树。
_Avoid_: 根消息, 空节点

**Delete (删除)**:
唯一的删除语义：剪除以目标节点为根的整棵子树。禁止摘除单个中间节点。子树内的分支名随叶子消亡；HEAD 若在子树内则回到被删节点的父节点；虚拟根不可删。
_Avoid_: 摘除, 剔除单点

**Fork (分叉)**:
一个节点获得第二个孩子的事件。仅由三种操作产生：重新生成 AI 回复（分叉点在用户节点）、编辑重发用户消息（分叉点在上一个 AI 节点，首条消息则在虚拟根）、从历史节点继续发送（分叉点在该节点）。配置变更本身永不产生分叉。
_Avoid_: 换模型分叉
